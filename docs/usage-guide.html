실행 전 확인

프로그램은 필요한 도구를 설치하지 않고, 이미 구성된 작업 환경을 확인해서 사용합니다.

Wwise Authoring

대상 프로젝트를 열고 WAAPI를 켠 뒤 설정창을 닫습니다.

Perforce CLI

일반 점검에는 p4.exe가 설치되어 있어야 합니다.

Perforce workspace

일반 점검에는 대상 프로젝트 폴더가 현재 workspace에 매핑되어 있어야 합니다.

쓰기 가능한 로컬 폴더

ZIP 전체를 로컬 폴더에 풀어야 설정과 보고서를 저장할 수 있습니다.

실행 파일만 따로 복사하지 마세요 WwiseOriginalsRelocator.exe와 _internal 폴더는 항상 함께 있어야 합니다.

빠른 시작

  1. ZIP 전체 압축 해제네트워크 드라이브가 아닌 쓰기 가능한 로컬 폴더를 권장합니다.
  2. 프로그램 실행WwiseOriginalsRelocator.exe를 더블 클릭합니다.
  3. 프로젝트 폴더 선택.wproj 파일이 있는 폴더를 선택합니다.
  4. 환경 확인 시작Wwise, WAV, Work Unit, Perforce, WAAPI 상태를 확인합니다.
  5. 이동 계획 만들기환경 확인이 모두 통과한 뒤 읽기 전용 계획을 만듭니다.
  6. 결과 검토이동 가능, 이미 정리됨, 담당자 확인 항목과 보고서 경로를 확인합니다.
  7. 파일 하나 선택사전 검증을 통과한 이동 가능 항목 하나만 선택합니다.
  8. 적용 또는 Rollback파일명과 경로를 다시 확인한 뒤 적용하고, 테스트 후 manifest로 복구합니다.

Perforce 없이 로컬 테스트

Perforce가 준비되지 않은 컴퓨터에서도 실제 Wwise 프로젝트와 로컬 파일을 읽어 GUI의 환경 확인 및 이동 계획 흐름을 체험할 수 있습니다.

  1. Wwise에서 테스트 프로젝트 열기WAAPI를 사용할 수 있는 상태로 프로젝트를 엽니다.
  2. 프로젝트 폴더 선택.wproj 파일이 있는 폴더를 선택합니다.
  3. Perforce 없이 로컬 테스트 켜기프로젝트 선택 영역의 테스트 스위치를 켭니다.
  4. 환경 확인과 이동 계획 실행Perforce 항목은 건너뛰고 Wwise, WAV, Work Unit과 로컬 경로를 확인합니다.
로컬 통과는 운영 준비 완료가 아닙니다이 모드는 Perforce CLI, workspace mapping, opened 상태를 검증하지 않습니다. 실제 운영 전에는 테스트 모드를 끄고 일반 환경 확인을 통과해야 합니다.
로컬 테스트에서는 변경 기능이 꺼집니다파일 선택, apply, rollback 버튼은 비활성화되며 Wwise와 로컬 경로를 읽는 흐름만 실행합니다.

환경 확인 결과 읽기

항목실패했을 때 확인할 것
프로젝트 폴더선택한 폴더가 존재하고 접근 가능한지 확인합니다.
Wwise 프로젝트선택한 폴더에 .wproj 파일이 정확히 하나 있는지 확인합니다.
Originals WAVOriginals 아래에 WAV source가 있는지 확인합니다.
Work Unit sourcesource reference가 포함된 .wwu 파일이 있는지 확인합니다.
Perforce CLI고급 연결 설정에서 p4.exe를 직접 선택합니다.
Perforce workspaceP4V에서 현재 workspace와 프로젝트 폴더 mapping을 확인합니다.
Wwise 연결대상 프로젝트가 열려 있고 WAAPI가 켜져 있는지, 설정창이나 확인창이 남아 있지 않은지 확인합니다.
WAAPI 연결은 자동으로 찾습니다기본 WAMP 8080을 먼저 확인하고, 사용할 수 없으면 Wwise HTTP WAAPI 8090을 자동으로 확인합니다. 연결된 Wwise 프로젝트도 선택한 폴더와 일치하는지 검사하므로, 기본 설정에서는 주소를 직접 바꿀 필요가 없습니다.

이동 계획 읽기

이동 가능

현재 경로와 제안 경로가 명확하고 사전 점검 대상이 된 source입니다.

이미 정리됨

source가 이미 Wwise 객체 분류와 같은 위치에 있습니다.

담당자 확인

source 공유, 경로 충돌, 분류 불명확 등 자동 판단할 수 없는 항목입니다.

사전 검증 확인 필요

누락 파일, workspace 밖 경로, 이미 열린 파일 같은 blocker가 있습니다.

계획 단계까지는 변경이 없습니다실제 변경은 사전 검증을 통과한 항목 하나를 선택하고 확인창에서 승인한 뒤에만 시작됩니다.

단일 파일 적용과 Rollback

프로그램은 한 번에 WAV 하나만 처리하며, 첫 Perforce 변경 전에 rollback manifest를 저장합니다.

  1. 이동 가능 항목 하나 선택계획 전체의 사전 검증이 통과해야 선택할 수 있습니다.
  2. 선택한 파일 적용확인창의 파일명과 경로를 검토하면 p4 edit, p4 move, Work Unit source 경로 교체가 실행됩니다.
  3. Wwise 다시 불러오기External Project Changes 안내에서 affected Work Unit을 다시 불러옵니다.
  4. P4V 검토WAV move와 Work Unit diff가 의도한 한 줄 경로 변경뿐인지 확인합니다.
  5. 테스트 후 Rollback이 파일 Rollback을 누르면 manifest에 기록된 WAV와 Work Unit만 원래 상태로 복구합니다.
프로그램은 submit하지 않습니다최종 제출은 팀의 기존 P4V 검토와 승인 절차를 사용하세요. 적용된 manifest가 남아 있는 동안에는 다른 파일을 추가 적용할 수 없습니다.

저장되는 Portable 데이터

설정, 로그, 보고서는 프로그램 폴더의 data 아래에만 저장됩니다.

WwiseOriginalsRelocator/ ├─ WwiseOriginalsRelocator.exe ├─ _internal/ ├─ 사용가이드.html ├─ LICENSE.txt ├─ VERSION.txt └─ data/ ├─ settings.json ├─ logs/relocator.log └─ reports/ └─ ...-apply/rollback-manifest.json

프로그램을 다른 PC로 옮길 때 기존 설정과 보고서가 필요하면 data도 함께 복사합니다. 초기화하려면 프로그램을 닫고 data를 별도로 보관한 뒤 삭제할 수 있습니다.

문제가 생겼을 때

창이 전혀 열리지 않음

_internal 폴더가 실행 파일과 함께 있는지 확인합니다. Windows 화면 구성 요소인 Edge WebView2 사용 가능 여부는 사내 IT 담당자에게 문의하세요. 이 프로그램은 WebView2를 자동 설치하지 않습니다.

Windows가 알 수 없는 앱이라고 표시함

개발용 artifact는 코드 서명이 없을 수 있습니다. 회사 보안 정책을 우회하거나 임의 예외를 추가하지 말고, 배포 담당자에게 승인된 서명본인지 확인하세요.

Wwise 연결을 통과하지 못함

Wwise에서 대상 프로젝트와 WAAPI가 활성화되어 있는지 확인하세요. 설정창이나 저장 확인창이 열려 있으면 닫고 환경 확인을 다시 실행합니다. 프로그램은 WAMP 8080에 다른 프로그램이 있어도 Wwise HTTP WAAPI 8090을 자동으로 확인합니다. 화면에 프로젝트 불일치가 표시되면 선택한 폴더의 프로젝트를 Wwise에서 여세요.

이동 계획 만들기가 오래 걸림

현재 버전은 WAAPI 연결과 source 조회에 제한 시간을 적용하며, 응답하지 않으면 오류 메시지로 돌아옵니다. 이전 버전에서 계속 대기 중이라면 프로그램을 닫고 최신 버전으로 다시 실행한 뒤 환경 확인부터 진행하세요.

적용 후 Wwise에 이전 경로가 보임

Wwise의 External Project Changes 안내에서 affected Work Unit을 다시 불러오세요. Wwise가 외부 변경을 다시 읽기 전에는 화면에 이전 source 경로가 남아 있을 수 있습니다.

지원 요청에 필요한 정보

  • 화면에 표시된 실패 항목과 메시지
  • data/logs/relocator.log
  • VERSION.txt

로그에는 내부 폴더 경로가 포함될 수 있으므로 외부 전달 전 회사 보안 정책을 확인하세요.

현재 버전의 안전 범위

  • 사전 검증을 통과한 WAV 하나만 p4 move합니다.
  • 선택한 Wwise 객체 GUID의 source 경로 한 곳만 수정합니다.
  • 첫 변경 전에 rollback manifest를 저장합니다.
  • Rollback은 manifest에 기록된 WAV와 Work Unit만 복구합니다.
  • p4 submit은 실행하지 않습니다.
  • Python, Wwise, Perforce 또는 다른 프로그램을 설치하지 않습니다.
