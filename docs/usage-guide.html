P4V connection

P4V 연결 사용

파일 명령은 p4.exe가 실행하지만, 서버·사용자·workspace와 기존 로그인 티켓은 P4V 환경을 그대로 사용합니다.

1. P4V에서 로그인

대상 서버와 workspace를 선택합니다.

2. 연결 불러오기

프로젝트 선택 시 실제 P4PORT를 확인하고, P4CLIENT가 없으면 프로젝트에 맞는 workspace 하나를 자동으로 찾습니다.

3. 환경 확인

서버 연결과 프로젝트 mapping이 모두 통과하는지 확인합니다.

P4PORTP4USERP4CLIENTP4CHARSET
P4V Tools 메뉴에서 실행하면 가장 정확합니다

Tools → Manage Tools에 앱을 외부 도구로 등록하면 P4V가 현재 연결 값을 앱에 전달합니다.

P4V 화면만으로 적용하지는 않습니다

p4.exe를 찾지 못하면 고급 연결 설정의 직접 선택으로 지정하세요. 비밀번호와 티켓은 저장하지 않습니다.

No Perforce required

Perforce 없이 로컬 테스트

실제 Wwise 프로젝트와 로컬 파일을 읽어 환경 확인과 이동 계획 화면을 체험할 수 있습니다.

할 수 있는 것

  • Wwise와 WAAPI 연결 확인
  • Originals WAV와 Work Unit 읽기
  • 이동 계획 및 보고서 생성
  • 로컬 경로 유효성 검증

할 수 없는 것

  • Perforce workspace mapping 확인
  • opened 상태와 diff 검증
  • 파일 Apply 실행
  • Rollback 및 P4V 인계
로컬 통과는 운영 준비 완료가 아닙니다

실제 작업 전에는 로컬 테스트를 끄고 일반 환경 확인을 다시 통과해야 합니다.

Read-only preview

이동 계획 읽기

상태 배지를 먼저 보고, 이동 가능한 항목의 현재 경로와 대상 경로를 비교하세요.

이동 가능

현재 위치와 제안 경로가 명확하며 사전 검증을 통과할 수 있습니다.

이미 정리됨

source가 Wwise 객체 분류와 같은 위치에 있어 변경하지 않습니다.

담당자 확인

source 공유, 경로 충돌 등 자동 판단할 수 없는 항목입니다.

확인 필요

누락 파일, workspace 밖 경로, 이미 열린 파일 같은 blocker입니다.

상태Source현재 위치이동 위치
이동 가능CH04_S102_WT_001.wavScenario/CH04/...Script/CH04/...
이미 정리됨CH04_CUT_010.wavCutscene/CH04/...변경 없음
Object Root는 계획을 만들 때 보정됩니다

입력한 상위 경로 아래에서 선택한 챕터와 지원 분류가 있는 단일 Root를 찾으면 고급 설정에 자동 저장합니다. 후보가 여러 개면 임의로 선택하지 않고 직접 확인하도록 안내합니다.

계획 단계까지는 변경이 없습니다

사전 검증을 통과한 파일을 선택하고 최종 확인창을 승인한 뒤에만 실제 변경이 시작됩니다.

다중 파일 안전 적용

선택 파일 적용과 Rollback

첫 Perforce 변경 전에 전체 rollback manifest를 저장하고, 선택한 파일을 같은 changelist에서 처리합니다.

1. 이동 가능 항목 선택

사전 검증이 모두 통과한 뒤 적용할 WAV를 하나 이상 선택합니다.

2. 선택 파일 적용

확인창의 전체 목록을 승인하면 p4 edit, p4 move와 정확한 Work Unit source 경로 변경을 실행합니다.

3. Wwise Reload 대기 확인

앱이 적용 완료로 확정하지 않고 Wwise Reload 필요 상태로 전환되는지 확인합니다.

4. Wwise 다시 불러오기

그 상태가 표시된 뒤 External Project Changes 안내에서 affected Work Unit을 선택하고 Reload를 누릅니다.

5. Wwise 반영 확인

Reload가 끝난 뒤 앱의 Wwise 반영 확인을 눌러 파일 위치, WWU 변경 범위, Perforce opened/diff와 live Wwise 객체를 검증합니다.

6. P4V 검토 및 인계

WAV move와 Work Unit diff를 최종 검토합니다. 프로그램은 submit하지 않습니다.

7. 마감 상태 확인

P4V에서 submit 또는 revert를 마친 뒤 열린 파일과 Wwise 결과를 확인하고 다음 작업을 허용합니다.

Rollback 경로: P4V 마감 전이라면 manifest에 기록된 WAV와 Work Unit만 역순으로 원래 상태로 복구할 수 있습니다.
프로그램은 submit하지 않습니다

최종 제출은 팀의 기존 P4V 검토와 승인 절차를 사용하세요. 현재 작업을 마감하거나 Rollback하기 전에는 다음 적용을 시작할 수 없습니다.

Operation history

최근 작업 기록

현재 프로젝트에서 앱으로 처리한 최근 작업을 최대 20개까지 보여줍니다. 조회만으로 Perforce 상태가 변경되지는 않습니다.

Wwise Reload 필요

파일 변경은 끝났지만 Wwise의 affected Work Unit Reload와 반영 확인이 남았습니다.

P4V 마감 대기

자동 검증을 통과하고 P4V의 submit 또는 revert를 기다립니다.

완료

P4V 마감과 Wwise 결과까지 확인되어 다음 작업을 시작할 수 있습니다.

Rollback 완료

manifest에 기록된 파일이 원래 상태로 복구되었습니다.

복구 필요

적용이나 자동 복구가 완전히 끝나지 않아 운영 담당자의 확인이 필요합니다.

Troubleshooting

문제가 생겼을 때

증상과 같은 항목을 펼쳐 순서대로 확인하세요.

운영체제가 확인되지 않은 앱이라고 표시함

개발용 artifact에는 Windows 코드 서명 또는 macOS notarization이 없을 수 있습니다. 보안 정책을 우회하지 말고 배포 담당자에게 승인된 빌드인지 확인하세요.

Wwise 연결을 통과하지 못함

대상 프로젝트와 WAAPI가 활성화되어 있는지 확인하고 설정창이나 저장 확인창을 닫으세요. 프로젝트 불일치가 표시되면 선택한 폴더의 프로젝트를 Wwise에서 다시 엽니다.

P4V는 연결되지만 Perforce 점검이 실패함

Workspace 선택 필요가 표시되면 P4V에서 이 프로젝트가 포함된 workspace를 선택한 뒤 고급 연결 설정 → 연결 불러오기를 누르세요. 서버 연결 실패와 workspace 미선택은 별도 항목으로 표시됩니다. 계속 연결을 찾지 못하면 p4.exe를 직접 선택하거나 P4V의 Tools → Manage Tools에 앱을 등록해 실행합니다.

이동 계획 만들기가 오래 걸림

현재 버전은 WAAPI 연결과 source 조회에 제한 시간을 적용합니다. 계속 대기한다면 앱을 닫고 최신 버전으로 다시 실행한 뒤 환경 확인부터 진행하세요.

적용 후 Wwise에 이전 경로가 보임

External Project Changes에서 affected Work Unit을 다시 불러온 뒤 Wwise 반영 확인을 누르세요. 계속 실패하면 보고서를 확인하고 P4V 마감 전 Rollback을 진행합니다.

Work Unit 기존 변경 때문에 적용이 차단됨

앱이 수정할 .wwu에 다른 로컬 변경이 감지된 상태입니다. 해당 변경을 먼저 별도 changelist에 정리하거나 submit/revert한 뒤 환경 확인과 이동 계획을 다시 실행하세요. 다른 작업을 보존한 채 강제로 덮어쓰지는 않습니다.

지원 요청에 필요한 정보
  • 화면에 표시된 실패 항목과 메시지
  • data/logs/relocator.log
  • VERSION.txt
  • 해당 작업의 data/reports 폴더

Safety boundaries

현재 버전의 안전 범위

자동화가 책임지는 범위와 사용자가 최종 확인해야 하는 경계를 분명하게 유지합니다.

선택 파일만 이동

사전 검증을 통과한 선택 WAV만 같은 changelist에서 처리합니다.

정확한 WWU 변경

선택한 Wwise 객체 GUID의 source 경로만 수정합니다.

Manifest 우선

첫 변경 전에 rollback manifest를 저장합니다.

기존 WWU 변경 보호

대상 Work Unit에 다른 로컬 변경이 있으면 첫 Perforce 변경 전에 적용을 차단합니다.

묶음 자동 복구

한 항목이라도 실패하면 이미 적용한 항목을 역순으로 복구합니다.

적용 후 재검증

로컬 파일, Perforce 상태와 live Wwise 객체를 다시 확인합니다.

P4V 마감 잠금

submit 또는 revert가 확인될 때까지 다음 적용을 막습니다.

자동 설치 없음

Python, Wwise, Perforce 또는 다른 프로그램을 설치하지 않습니다.

자동 submit 없음

p4 submit은 실행하지 않으며 최종 결정은 P4V에 남깁니다.
